1. Используемые технологии - PHP 8.1, MySQL 8.0 (можно не менее PHP 8.0, MySQL 5.5)
2. Используется фреймворк phalcon 5.8.0. Инструкция по установке находится по адресу https://docs.phalcon.io/5.8/installation/#installation-platforms
3. Для настройки надо заполнить настройки подключения в MySQL в файле config/config.php в секции database. Остальные настройки опциональны

Все поля в запросах, кроме отмеченных, обязательны к заполнению

Ответ API в формате JSON. В ответе всегда есть поле "success". Если оно "true" - запрос успешно выполнился, если "false" - запрос завершился ошибкой, описание ошибки находится в поле "message". В запросе с ошибкой всегда только 2 поля - "success" и "false"

После авторизации клиент получает токен, который надо использовать в дальнейших запросах - надо во все запросы, относящиеся к постам, добавить header "Authorization", а значение - токен

Ответы на GET-запросы к постам, категориям и изображениям представляют из себя 2 секции - блок с запрашиваемыми данными и блок с данными пагинации (см. файл Запросы)
В этих запросах используются следующие параметры:
- expand - вывод данных из связаных моделей (например, в запросе данных категорий при указании "posts" в параметре выводится блок "posts" как при запросе "/api/v1/posts"). Если надо указать несколько связанных моделей, то они перечисляются через запятую
- fields - перечисление только тех полей, которые надо вывести в ответе. Не рапространяется на связаные модели (expand). Если надо указать несколько полей, то они перечисляются через запятую
- sort - сортировка данных по запрашиваемой модели. Синтаксис: {имя_поля}:{направление сортировки}. Если направление прямое, то частью ":{направление сортировки}" можно пренебречь. Например: "title" и "id:DESC". Если надо указать несколько полей сортировки, то они перечисляются через запятую

Для создания нового пользователя через консоль надо выполнить команду:
php /app/Cli/cli.php {login} {password} {group_id}
где {login} - логин (string) - обязательное поле
{password} - пароль (string) - обязательное поле
{group_id} - ID группы пользователей(1 - админ, 2 - автор) (int) - необязательное поле, если не указано, то устанавливается "автор"

Чтобы развернуть базу, использую миграцию, надо выполнить комманду
php /app/Cli/cli.php migration run
